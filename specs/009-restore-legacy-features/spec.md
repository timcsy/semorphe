# Feature Specification: 整合舊版已驗證功能至新架構

**Feature Branch**: `009-restore-legacy-features`
**Created**: 2026-03-06
**Status**: Draft
**Input**: 將舊版 code-blockly 中已驗證的好功能整合進新架構，涵蓋程式執行、雙向高亮、積木互動改進、輔助功能四大類。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 程式執行與即時回饋 (Priority: P1)

學習者在積木編輯器中組裝好一段程式（例如用積木拼出一個求和迴圈），點擊「執行」按鈕後，右側程式碼編輯器下方的底部面板區自動切換到 Console 分頁並顯示程式輸出結果。如果程式需要使用者輸入（如 `cin >> n`），Console 分頁會顯示輸入提示，學習者輸入值後程式繼續執行。學習者也可以選擇「逐步執行」模式，每按一次只執行一個步驟，底部面板自動切換到變數監看分頁，即時看到每個變數的名稱、型別和當前值，變更的值會以高亮標示。底部面板區以分頁標籤切換 Console 和變數監看，僅佔右半邊，左側積木編輯器不受影響。

**Why this priority**: 程式執行是學習程式設計的核心體驗。沒有執行能力，工具只是一個「畫積木」的玩具，無法驗證學習者的理解是否正確。這是將工具從「展示」升級為「學習」的關鍵功能。

**Independent Test**: 可透過建立簡單積木程式（如印出 "Hello"）、點擊執行、確認 Console 顯示輸出來獨立驗證。不依賴其他新功能。

**Acceptance Scenarios**:

1. **Given** 工作區有一組積木程式（含 print 語句）且已同步，**When** 學習者點擊「執行」按鈕，**Then** Console 分頁展開並顯示程式輸出文字
8. **Given** 積木已被修改但尚未同步，**When** 學習者點擊「執行」，**Then** 系統顯示提示「積木已變更，是否先同步再執行？」，學習者可選擇同步後執行或直接執行當前版本
2. **Given** 程式包含 input 語句，**When** 執行到 input 時，**Then** Console 面板顯示內聯輸入欄位，學習者輸入值後程式繼續
3. **Given** 學習者選擇「逐步執行」，**When** 每按一次 Step 按鈕，**Then** 執行一個步驟，當前執行的積木被高亮，變數面板顯示該步驟後所有變數的最新值
4. **Given** 程式正在執行中，**When** 學習者點擊「暫停」，**Then** 執行暫停，可繼續或停止
5. **Given** 程式正在執行中，**When** 學習者點擊「停止」，**Then** 執行立即中止，所有高亮清除，Console 顯示「已停止」
6. **Given** 學習者選擇不同執行速度（慢/中/快），**When** 使用逐步或動畫執行，**Then** 步驟之間的間隔時間符合所選速度
7. **Given** 程式執行發生錯誤（如除以零），**When** 錯誤發生，**Then** Console 顯示錯誤訊息，執行停止，狀態顯示為「錯誤」

### User Story 2 - 雙向高亮與同步方向提示 (Priority: P2)

學習者在積木編輯器中點選一個積木時，右側程式碼編輯器中對應的程式碼行被高亮標示。反之，學習者在程式碼編輯器中將游標移到某一行時，左側對應的積木被高亮。此外，當學習者修改了積木但尚未同步到程式碼時，「積木→程式碼」按鈕會顯示視覺提示（如微光或顏色變化），反之亦然。

**Why this priority**: 雙向高亮直接體現了「程式碼和積木是同一棵語義樹的投影」這一核心概念。對初學者而言，看到積木和程式碼的即時對應關係是建立程式設計心智模型的重要橋樑。

**Independent Test**: 可透過點選積木、確認程式碼高亮，以及反向操作來獨立驗證。不依賴程式執行功能。

**Acceptance Scenarios**:

1. **Given** 工作區有積木且已同步程式碼，**When** 學習者點選一個積木，**Then** 右側程式碼編輯器中對應的行被高亮
2. **Given** 工作區有積木且已同步程式碼，**When** 學習者在程式碼編輯器中移動游標到某行，**Then** 左側對應的積木被高亮
3. **Given** 學習者修改了積木（拖拉/刪除/新增），**When** 尚未點擊同步按鈕，**Then** 「積木→程式碼」按鈕顯示視覺提示
4. **Given** 學習者在程式碼編輯器中修改了程式碼，**When** 尚未點擊同步按鈕，**Then** 「程式碼→積木」按鈕顯示視覺提示
5. **Given** 同步完成後，**When** 兩側內容一致，**Then** 兩個同步按鈕的視覺提示消失

### User Story 3 - 進階積木互動 (Priority: P3)

學習者使用條件判斷積木時，可以透過齒輪圖示或加減按鈕新增 else-if 分支和 else 區塊，組合出多層條件邏輯。使用變數參考積木時，下拉選單自動列出工作區中已宣告的所有變數名稱。使用輸入積木時，可以一次讀取多個變數（如 `cin >> a >> b >> c`）。編輯過程中，系統自動偵測積木配置問題（如缺少必要連接、型別不匹配）並在積木上顯示警告圖示。

**Why this priority**: 這些功能提升積木編輯的表達力和易用性，但不影響基本的建立和執行流程。初學者可以先用簡單積木學習，進階後才需要 else-if 鏈和多變數輸入。

**Independent Test**: 可透過拖出 if 積木、使用齒輪新增 else-if、驗證積木結構變化來獨立測試。

**Acceptance Scenarios**:

1. **Given** 學習者拖出一個 if 積木，**When** 使用齒輪或加減按鈕新增 else-if 分支，**Then** 積木動態增加一個 else-if 條件區和執行區
2. **Given** 學習者已新增多個 else-if 分支，**When** 使用減號按鈕移除一個分支，**Then** 該分支被移除，其餘分支不受影響
3. **Given** 工作區中有多個已宣告的變數，**When** 學習者點擊變數參考積木的下拉選單，**Then** 選單列出所有可用的變數名稱
4. **Given** 學習者拖出 input 積木，**When** 使用加號按鈕新增輸入變數，**Then** 積木增加一個變數欄位，最終生成 `cin >> a >> b` 形式的程式碼
5. **Given** 積木配置有問題（如 if 積木缺少條件），**When** 系統執行診斷，**Then** 該積木顯示警告圖示與提示文字

### User Story 4 - 輔助功能與擴充性 (Priority: P4)

學習者可以在積木編輯器上方看到一列快速存取按鈕，點擊即可在工作區中央建立常用積木（如變數宣告、if、迴圈、print），免去從工具箱拖拉的步驟。在程式碼編輯器中，學習者可以點擊行號旁的區域設置斷點，配合逐步執行使用。進階使用者可以上傳自訂積木定義檔案來擴充工具箱。所有操作的回饋（成功、失敗、警告）透過短暫的 Toast 通知呈現。

**Why this priority**: 這些是提升效率和擴充性的輔助功能，對核心學習流程非必要，但能顯著改善日常使用體驗。

**Independent Test**: 可透過點擊快速存取按鈕、確認積木出現在工作區來獨立測試。

**Acceptance Scenarios**:

1. **Given** 快速存取列可見，**When** 學習者點擊「變數」快速按鈕，**Then** 一個變數宣告積木出現在工作區中央
2. **Given** 程式碼編輯器有內容，**When** 學習者點擊某行的行號旁區域，**Then** 該行顯示斷點標記（圓點圖示）
3. **Given** 已設置斷點且使用逐步執行，**When** 執行到斷點行，**Then** 執行自動暫停
4. **Given** 進階使用者有一個自訂積木 JSON 檔案，**When** 透過上傳按鈕匯入，**Then** 新積木出現在工具箱中，可正常使用
5. **Given** 學習者執行任何操作（匯出、匯入、上傳積木），**When** 操作完成，**Then** 顯示短暫的 Toast 通知（成功為綠色、失敗為紅色），數秒後自動消失

---

### Edge Cases

- 程式執行陷入無窮迴圈時，系統在超過最大步數限制（預設 100,000 步）後自動中止，Console 顯示逾時警告
- 逐步執行時學習者修改了積木，系統提示「程式已變更，請重新執行」並停止當前執行
- 同步方向提示在頁面載入恢復狀態後正確反映兩側的一致性
- 變數監看面板在遞迴函式中正確顯示各層作用域的變數
- 快速存取按鈕在不同認知層級（L0/L1/L2）只顯示該層級可用的積木
- 自訂積木上傳格式錯誤時，顯示明確的錯誤訊息並保持現有工具箱不變
- 斷點設置在程式碼因同步更新而變化時，盡可能保留在對應的語義位置
- Console 面板在大量輸出時保持效能，自動滾動到最新輸出

## Requirements *(mandatory)*

### Functional Requirements

**程式執行**:
- **FR-001**: 系統 MUST 提供「執行」功能，將當前語義樹交給直譯器執行，並將標準輸出顯示在 Console 面板
- **FR-002**: 系統 MUST 提供「逐步執行」功能，每次執行一個語義節點，並在每步後更新變數面板和積木/程式碼高亮
- **FR-003**: 系統 MUST 提供「暫停」和「停止」功能，在執行過程中可隨時暫停或中止
- **FR-004**: 系統 MUST 提供執行速度選擇，至少支援慢速（800ms）、中速（300ms）、快速（50ms）三個等級
- **FR-005**: 系統 MUST 在程式需要標準輸入時，在 Console 面板提供內聯輸入機制
- **FR-006**: 系統 MUST 在執行錯誤時顯示有意義的錯誤訊息，包含錯誤類型和發生位置
- **FR-006a**: 系統 MUST 在偵測到積木或程式碼已修改但未同步時，於點擊「執行」時顯示提示，讓學習者選擇同步後執行或直接執行當前版本

**Console 面板**:
- **FR-007**: 系統 MUST 在右側程式碼編輯器下方提供底部面板區，以分頁標籤切換 Console 和變數監看面板
- **FR-008**: Console 面板 MUST 在有新輸出時自動展開並滾動到最新內容
- **FR-009**: Console 面板 MUST 提供清除功能

**變數監看面板**:
- **FR-010**: 系統 MUST 提供可收合的變數監看面板，以表格形式顯示變數的名稱、型別和值
- **FR-011**: 變數面板 MUST 在每步執行後更新，並高亮標示值有變化的變數
- **FR-012**: 變數面板 MUST 正確反映當前作用域中可見的變數，當前作用域的變數直接顯示，父作用域以可收合的分組呈現（如「全域」「main」「for 迴圈」）

**雙向高亮**:
- **FR-013**: 系統 MUST 在學習者選取積木時，高亮程式碼編輯器中對應的行
- **FR-014**: 系統 MUST 在學習者移動程式碼游標時，高亮對應的積木
- **FR-015**: 系統 MUST 在同步後建立積木與程式碼行之間的映射關係

**同步提示**:
- **FR-016**: 系統 MUST 在任一側內容被修改且尚未同步時，在對應的同步按鈕上顯示視覺提示
- **FR-017**: 同步完成後視覺提示 MUST 自動消失

**積木互動改進**:
- **FR-018**: if 積木 MUST 支援動態新增/移除 else-if 分支和 else 區塊
- **FR-019**: 變數參考積木 MUST 自動掃描工作區中已宣告的變數並提供下拉選單
- **FR-020**: 輸入積木 MUST 支援動態新增/移除多個變數欄位
- **FR-021**: 系統 MUST 提供自動診斷功能，偵測積木配置問題並顯示警告

**輔助功能**:
- **FR-022**: 系統 MUST 在積木編輯器上方提供快速存取列，包含常用積木的一鍵建立按鈕
- **FR-023**: 快速存取列的按鈕 MUST 依當前認知層級過濾
- **FR-024**: 程式碼編輯器 MUST 支援在行號旁設置和移除斷點
- **FR-025**: 系統 MUST 支援匯入自訂積木定義檔案，並將新積木加入工具箱
- **FR-026**: 系統 MUST 以 Toast 通知形式回饋操作結果（成功/失敗/警告），通知在數秒後自動消失

### Key Entities

- **執行會話 (Execution Session)**: 一次程式執行的完整生命週期，包含狀態（idle/running/paused/stopped/error）、步驟記錄、累積輸出
- **步驟快照 (Step Snapshot)**: 單一執行步驟的狀態，包含當前語義節點位置、變數作用域快照、輸出增量
- **Source Mapping**: 積木 ID 與程式碼行範圍的對應關係，每筆記錄包含 blockId、startLine、endLine
- **診斷結果 (Diagnostic)**: 積木配置問題的描述，包含嚴重程度、相關積木 ID、訊息文字
- **快速存取項目 (Quick Access Item)**: 快速按鈕的定義，包含積木類型、圖示、標籤、認知層級門檻

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 學習者可以在 2 次點擊內（執行按鈕 + 確認輸出）完成一個簡單程式的執行與結果驗證
- **SC-002**: 逐步執行時，每步的變數面板更新和積木高亮在 200 毫秒內完成，使用者感受為即時
- **SC-003**: 雙向高亮在選取積木或移動程式碼游標後 100 毫秒內完成對應側的高亮顯示
- **SC-004**: 100% 的同步按鈕狀態變化正確反映兩側的一致性（無誤報或漏報）
- **SC-005**: else-if 分支可動態新增至少 10 個，且每次新增/移除操作不超過 500 毫秒
- **SC-006**: 變數參考下拉選單在任何工作區狀態下都只列出合法的已宣告變數，0 個遺漏或錯誤
- **SC-007**: 快速存取按鈕建立積木的操作在 300 毫秒內完成，積木出現在工作區可視範圍內
- **SC-008**: 自訂積木上傳後，新積木在 1 秒內出現在工具箱且可正常拖出使用
- **SC-009**: 所有新功能不破壞現有的雙向同步管線——既有的「積木到程式碼」和「程式碼到積木」測試 100% 通過
- **SC-010**: 無窮迴圈保護在達到步數上限時自動中止，Console 在 1 秒內顯示逾時警告

## Assumptions

- 舊版的 SemanticInterpreter、StepController、ConsolePanel、VariablePanel 程式碼可作為參考，但需依新架構的語義樹結構重新整合
- Console 面板和變數面板的 UI 定義檔案已存在於新版 src/ui/panels/ 目錄下，但尚未被主 App 使用
- 程式執行基於語義樹直譯（不編譯為機器碼），適用於教學場景的程式規模
- 執行步數上限預設為 100,000 步，可在未來配置化
- 斷點功能需要程式碼編輯器支援 gutter 互動（目前使用 Monaco Editor）
- 快速存取列的積木項目可在未來透過配置檔調整，不寫死在程式碼中
- Toast 通知持續時間預設為成功 2 秒、錯誤 4 秒

## Clarifications

### Session 2026-03-06

- Q: Console 與變數面板的 UI 配置位置？ → A: 放在右側程式碼編輯器下方的底部面板區，Console 和變數面板以分頁標籤切換，僅佔右半邊，左側積木編輯器不受影響
- Q: 點擊「執行」時是否自動同步？ → A: 智慧提示模式——偵測到未同步時顯示提示「積木已變更，是否先同步再執行？」，學習者可選擇同步後執行或直接執行當前版本
- Q: 變數監看面板的作用域顯示策略？ → A: 當前作用域的變數直接顯示，父作用域以可收合的分組呈現（如「全域」「main」「for 迴圈」）

## Scope Boundaries

**包含**:
- 程式執行（直譯器整合、Console、變數監看、執行控制）
- 雙向高亮與 Source Mapping
- 同步方向提示
- else-if 鏈支援、變數名自動收集、多變數輸入、診斷系統
- 快速存取列、斷點、自訂積木上傳、Toast 通知

**不包含**:
- 編譯為原生程式碼或 WebAssembly
- 多人協作或即時共編
- 程式碼自動補全或智慧提示
- 效能分析器或記憶體追蹤
- 新增程式語言支援（僅 C++）
- 版本控制或歷史記錄功能
