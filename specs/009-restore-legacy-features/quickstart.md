# Quickstart: 009-restore-legacy-features

## 驗證場景

### 場景 1: 基本程式執行

1. 在積木編輯器拖出：宣告 int x = 5 → 輸出 x
2. 點擊「積木→程式碼」同步
3. 點擊「執行」按鈕
4. **預期**: Console 分頁自動顯示，輸出 `5`，狀態顯示「已完成」

### 場景 2: 逐步執行與變數監看

1. 建立積木：宣告 int x = 0 → while (x < 3) { x = x + 1 } → 輸出 x
2. 同步後點擊「逐步」按鈕
3. 每按一次 Step，觀察：
   - 當前執行的積木被高亮
   - 變數面板分頁顯示 x 的值逐步從 0 → 1 → 2 → 3
   - 變更的值有高亮標示
4. **預期**: 3 次迴圈後輸出 3，狀態顯示「已完成」

### 場景 3: stdin 互動

1. 建立積木：輸入 x → 輸出 x
2. 執行程式
3. **預期**: Console 顯示輸入提示，輸入 `42` 後按 Enter，Console 顯示 `42`

### 場景 4: 雙向高亮

1. 建立多行積木程式並同步
2. 在積木編輯器中點選某個積木
3. **預期**: 右側程式碼編輯器對應的行被高亮
4. 在程式碼編輯器中將游標移到另一行
5. **預期**: 左側對應的積木被高亮

### 場景 5: 同步提示

1. 拖入一個新積木
2. **預期**: 「積木→程式碼」按鈕出現視覺提示
3. 點擊同步
4. **預期**: 視覺提示消失

### 場景 6: else-if 積木

1. 拖出 if 積木
2. 使用 + 按鈕新增 else-if 分支
3. **預期**: 積木增加一個 else-if 條件區和執行區
4. 使用 + 按鈕新增 else 區塊
5. **預期**: 積木增加 else 執行區

### 場景 7: 變數參考下拉

1. 建立 int x = 5 和 int y = 10 兩個宣告積木
2. 拖出變數參考積木
3. 點擊下拉選單
4. **預期**: 選單列出 x 和 y

### 場景 8: 快速存取列

1. 觀察積木編輯器上方的快速存取列
2. 點擊「變數」按鈕
3. **預期**: 一個變數宣告積木出現在工作區中央

### 場景 9: 無窮迴圈保護

1. 建立 while (true) {} 積木程式
2. 執行
3. **預期**: 執行在數秒內自動中止，Console 顯示逾時警告

### 場景 10: 未同步時執行

1. 修改積木但不同步
2. 點擊「執行」
3. **預期**: 顯示提示「積木已變更，是否先同步再執行？」
